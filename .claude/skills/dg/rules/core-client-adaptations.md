# Client Adaptations

Client-specific standards and preferences for discussion guide design.

## VFC / Whitney Standards

VFC prefers exploratory, conversational guides over comprehensive diagnostic tools.

### Structure
- 5-6 sections maximum
- Remove operational scaffolding (materials checklists, detailed time allocations)
- Linear narrative flow: attraction → engagement → belonging → future
- Universal questions (avoid segment-specific split paths)

### Question Approach

| Prefer | Avoid |
| ------ | ----- |
| Conversational | Diagnostic |
| Emotional | Transactional |
| General | SKU-specific |
| Exploratory | Comprehensive |

**Conversational over diagnostic:**
- Avoid structured exercises, forced rankings
- Allow organic conversation flow
- Questions feel like natural follow-ups, not interrogation

**Emotional over transactional:**
- Focus on feelings, belonging, experience quality
- Less emphasis on price points, feature comparisons
- "How did that make you feel?" over "What did you pay?"

**General over specific:**
- No SKU drilling
- Avoid detailed cost structures
- Category-level exploration over product-level detail

**Exploratory over comprehensive:**
- Allow conversation to go where it goes
- Not every objective needs explicit questions
- Trust moderator to pursue interesting threads

### What to Avoid

- Segment-specific deep dives (unless critical)
- Detailed purchase conversion funnels
- Economic modeling questions (costs, pricing, financial incentives)
- Extensive moderator guidance notes
- Post-session documentation protocols

### Sample Narrative Flow for VFC

1. **Attraction** — First encounter, initial draw
2. **Engagement** — Deepening relationship, key touchpoints
3. **Belonging** — Community, identity, loyalty signals
4. **Future** — Aspirations, unmet needs, evolution

## Adding New Client Adaptations

When a client has consistent preferences across projects, document here:

### Template
```markdown
## [Client Name] Standards

[Client] prefers [high-level description].

### Structure
- [Structural preferences]

### Question Approach
| Prefer | Avoid |
| ------ | ----- |
| ... | ... |

### What to Avoid
- [Specific anti-patterns]
```

### Triggering Client Adaptations

Check for client-specific rules when:
1. `PROJECT.md` mentions the client
2. `00-status.md` links to client page
3. User explicitly mentions client in brief

Apply adaptations as overlays to standard methodology.

# Jobs to Be Done Reference

Extract jobs-to-be-done from qualitative research using the JTBD framework.

## JTBD Framework

| Type | Definition | Example |
|------|------------|---------|
| **Functional** | Practical tasks to complete | "Quickly satisfy hunger between meetings" |
| **Emotional** | How people want to feel | "Feel like I'm treating myself without guilt" |
| **Social** | How people want to be perceived | "Show my kids I care about their nutrition" |

## Job Statement Format

```
When [situation], I want to [motivation], so I can [expected outcome]
```

## Extraction Process

1. Identify situations/contexts where needs arise
2. Surface the underlying motivation (not the product)
3. Connect to the desired outcome
4. Classify as functional, emotional, or social

## Prioritization

Rate each job by:
- **Frequency**: How often does this job arise?
- **Importance**: How critical is getting this job done?
- **Satisfaction**: How well are current solutions doing?

High-frequency + high-importance + low-satisfaction = opportunity

## Quality Benchmarks

**Weak Jobs (avoid)**:
- "I want a good product" (too vague)
- "I want to buy cookies" (product-focused, not job-focused)
- "I want to feel happy" (generic emotion)

**Strong Jobs (aim for)**:
- "When I'm exhausted after work but kids need snacks, I want to feel like a caring parent without cooking effort, so I can have energy left for quality time"
- "When hosting friends unexpectedly, I want to serve something that looks thoughtfully chosen, so I can seem like I have my life together"
- "When my diet is slipping, I want small indulgences that feel controlled, so I can enjoy without derailing my progress"

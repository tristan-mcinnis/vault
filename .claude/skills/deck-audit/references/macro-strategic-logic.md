# Macro: Strategic Logic

Evaluate the deck's overall narrative and argument structure. This is the most important level—a deck with weak macro logic cannot be saved by beautiful slides.

## Horizontal Logic

**Definition**: The slide titles, read in sequence, should tell the complete story.

### Test Method

1. List all slide titles in order
2. Read them aloud as a connected narrative
3. Ask: Does this make sense to someone who hasn't seen the slides?

### Scoring

| Rating | Criteria |
|--------|----------|
| **PASS** | Titles form coherent narrative; logical progression clear; no gaps |
| **WARN** | Mostly coherent; 1-2 unclear transitions; story recoverable |
| **FAIL** | Titles don't connect; random sequence; story unclear from titles |

### Common Failures

- **Topic labels**: "Market Overview" → "Competitive Analysis" → "Recommendations" (says what's there, not what it means)
- **Logical gaps**: Slide 4 discusses problem, Slide 5 jumps to solution (missing the "what we learned" bridge)
- **Orphan slides**: A slide that doesn't connect to before or after
- **Repetition**: Same point made in different words across multiple titles

### Strong vs Weak Title Sequences

**Weak** (topic labels):
1. Executive Summary
2. Market Context
3. Competitive Landscape
4. Consumer Research
5. Key Findings
6. Recommendations

**Strong** (action titles telling a story):
1. Premium cookies are under-penetrated in the Asian market
2. Local players dominate through ritual positioning—not taste
3. Western brands compete on ingredients, missing the cultural frame
4. Consumers want "earned indulgence"—premium treats for milestone moments
5. Three positioning territories can unlock the ritual occasion
6. Pilot in Singapore with the "celebration moments" campaign

## SCQA Structure

**Definition**: The deck should follow a Situation-Complication-Question-Answer arc, even if not explicitly labeled.

### The Four Elements

| Element | Purpose | Typical Location |
|---------|---------|------------------|
| **Situation** | Context everyone agrees on | Slides 1-3 |
| **Complication** | The tension, change, or problem | Slides 3-5 |
| **Question** | What we need to answer | Explicit or implied by slide 5-6 |
| **Answer** | Our insight and recommendations | Slides 6+ |

### Evaluation Criteria

| Element | Strong | Weak |
|---------|--------|------|
| **Situation** | Specific, grounded in data, genuinely uncontroversial | Generic, could apply to any company, already contains the problem |
| **Complication** | Creates genuine tension, makes audience uncomfortable, explains why now | Vague ("things are changing"), no urgency, complication is too obvious |
| **Question** | Specific, answerable by this deck, focuses attention | Too broad, rhetorical, doesn't match what follows |
| **Answer** | Directly addresses question, leads with insight, builds to recs | Doesn't match question, buries the insight, jumps to tactics |

### Diagnosing SCQA Problems

| Symptom | Likely Problem |
|---------|----------------|
| Deck feels flat, no energy | Weak or missing Complication |
| Audience asks "so what?" | Answer doesn't match Question |
| Recommendations feel random | No clear Question to anchor them |
| First 10 slides feel like throat-clearing | Situation too long or unfocused |
| Deck feels confrontational immediately | Missing Situation (jumped to Complication) |

## Executive Summary Alignment

**Definition**: The exec summary should be a standalone document that accurately previews the deck's argument.

### Test Method

1. Read the exec summary in isolation
2. Read the deck
3. Ask: Did the summary prepare me for what I just read?

### Common Misalignments

| Problem | Description |
|---------|-------------|
| **Bait and switch** | Summary promises insights the deck doesn't deliver |
| **Buried lede** | Key insight is in deck but not in summary |
| **Scope mismatch** | Summary covers things the deck doesn't address |
| **Stale summary** | Summary was written first and never updated |
| **Too vague** | Summary uses generalities that could apply to any project |

### Exec Summary Checklist

- [ ] States the core insight in the first paragraph
- [ ] Previews the main supporting points
- [ ] Includes the key recommendation(s)
- [ ] Matches the slide content in substance, not just topic
- [ ] Could stand alone as a 1-page brief

## Argument Progression

**Definition**: The deck should build logically from problem to understanding to action.

### Standard Consulting Flow

```
Context → Problem → Analysis → Insight → Implications → Recommendations → Next Steps
```

### Progression Patterns

| Pattern | When to Use | Watch Out For |
|---------|-------------|---------------|
| **Problem-first** | Audience knows there's an issue | Dwelling too long on problem before pivoting to solution |
| **Insight-first** | Confident audience, need to grab attention | Must still prove the insight with evidence |
| **Evidence-first** | Skeptical audience, need to build case | Can feel slow; risk losing attention before payoff |
| **Recommendation-first** | Time-constrained, trusted advisor | Audience may resist if they haven't bought in |

### Logical Flow Tests

1. **Dependency test**: Does each section require the previous section to make sense?
2. **Skip test**: If you skipped a section, would the argument still work? (If yes, maybe it's unnecessary)
3. **Sequence test**: Could the sections be reordered? (If yes, sequence may be arbitrary)

### Common Flow Problems

| Problem | Description | Fix |
|---------|-------------|-----|
| **Circular logic** | Evidence for insight is just the insight restated | Find independent supporting data |
| **Missing bridge** | Jump from data to recommendation | Add "what this means" synthesis |
| **Appendix dependency** | Key evidence in appendix, not core deck | Pull critical evidence forward |
| **Recommendation pile-up** | 10+ recommendations with no prioritization | Tier into P1/P2/P3 |
| **Dangling sections** | Content that doesn't connect to argument | Cut or clarify connection |

## Insight Visibility

**Definition**: The central insight should be unmistakably clear within the first 5 slides (or by the exec summary).

### The "Dinner Party Test"

If someone asks what the deck was about, could you answer in one sentence with the actual insight—not just the topic?

**Topic answer**: "It was about premium cookie positioning in Asia."
**Insight answer**: "Premium cookies in Asia need to own 'earned indulgence'—rewarding yourself for milestone moments—not compete on ingredients."

### Evaluating Insight Quality

| Dimension | Strong | Weak |
|-----------|--------|------|
| **Specificity** | "Consumers want X because Y" | "Consumers want better products" |
| **Non-obvious** | Challenges conventional wisdom | Says what everyone already knows |
| **Actionable** | Points to specific strategic moves | Could mean anything |
| **Supported** | Evidence is in the deck | Asserted without proof |
| **Memorable** | Sticks after the deck is closed | Forgettable |

### Insight Red Flags

- Insight is buried on slide 15
- Multiple "insights" compete for attention
- Insight is stated but not proven
- Insight doesn't connect to recommendations
- Insight is the question, not the answer ("We need to understand consumers better")

# Meso: Slide Architecture

Evaluate individual slides for clarity, focus, and contribution to the argument.

## One Idea Per Slide

**Definition**: Each slide should make exactly one clear point. If a slide is making multiple unrelated points, it should be split.

### Test Method

1. Ask: What is this slide's single point?
2. If you need "and" to answer, the slide is overloaded
3. Check: Do all elements (title, content, visuals) support that one point?

### Symptoms of Overloaded Slides

| Symptom | Description |
|---------|-------------|
| **Conjunction title** | "Market is growing AND competition is increasing" |
| **Unrelated bullets** | First bullet about consumers, third about operations |
| **Split focus** | Left side discusses one topic, right side another |
| **Competing visuals** | Two charts showing different, unconnected data |
| **"Also" syndrome** | Bullets that start with "Also," "Additionally," "Furthermore" |

### When Multiple Points Are Acceptable

- **Summary slides**: Explicitly synthesizing prior content
- **Comparison slides**: Two things compared on same dimensions
- **Process slides**: Steps in a sequence (but they're connected)
- **Framework slides**: Components of a single model

## Action Titles

**Definition**: Slide titles should be claims or takeaways, not topic labels.

### Topic Labels vs Action Titles

| Topic Label (Weak) | Action Title (Strong) |
|-------------------|----------------------|
| Market Overview | Premium cookies are under-penetrated in Asia |
| Consumer Insights | Consumers seek "earned indulgence" not everyday treats |
| Competitive Analysis | Local players win by owning ritual occasions |
| Recommendations | Three positioning territories unlock growth |
| Next Steps | Singapore pilot should launch in Q3 |

### Action Title Checklist

- [ ] States a claim, not a topic
- [ ] Could be disagreed with
- [ ] Makes sense without reading the slide body
- [ ] Is specific to this content, not generic
- [ ] Uses active verbs where possible

### Title Writing Tips

1. **Finish the sentence**: "This slide shows that..." → complete it → that's your title
2. **Add "so what"**: Take your topic label, add "...which means..." → that's closer
3. **Be specific**: Replace "significant" with a number, replace "growing" with a trend
4. **Avoid verbs of discovery**: "We found that..." is weak; just state the finding

### Acceptable Topic Labels

Some slides legitimately use topic labels:
- Agenda slides
- Section dividers (if the section title tells the story)
- Title slides
- Appendix headers

## What-So What-Now What

**Definition**: Each content slide should flow from evidence (what) to insight (so what) to implication (now what).

### The Flow

```
WHAT (the evidence)
    ↓
SO WHAT (what it means)
    ↓
NOW WHAT (what to do about it)
```

### Application by Slide Type

| Slide Type | What | So What | Now What |
|------------|------|---------|----------|
| **Data slide** | The chart/numbers | The insight callout | The implication for strategy |
| **Quote slide** | The verbatim quote | What it reveals | How to respond |
| **Case study** | What happened | Why it matters | What to take forward |
| **Framework** | The model components | What it explains | How to use it |

### Common Problems

| Problem | Description | Fix |
|---------|-------------|-----|
| **Orphan data** | Chart with no interpretation | Add callout box with "so what" |
| **Missing evidence** | Claim with no supporting data | Add the "what" that supports it |
| **Stopped at so what** | Insight stated but no implication | Add "This means we should..." |
| **All three crammed in** | Too dense, nothing lands | Spread across slides or cut |

## Slide Necessity Test

**Definition**: Every slide should earn its place in the deck. If it doesn't advance the argument, cut it.

### Questions to Ask

1. **What does this slide add that previous slides didn't?**
2. **If I cut this slide, would anyone notice the gap?**
3. **Is this slide for my argument or for my comfort?** (showing you did the work)
4. **Could this be an appendix slide?**

### Slides That Often Fail This Test

| Slide Type | When to Cut |
|------------|------------|
| **Methodology** | Unless audience will question credibility |
| **Detailed agenda** | Unless deck is 50+ slides |
| **Acknowledgments** | Almost always cut |
| **"What we did"** | Unless process itself is the insight |
| **Context everyone knows** | Unless challenging an assumption |
| **Every competitor profile** | Keep 2-3 exemplars, cut the rest |

### Slides Worth Keeping

- Directly supports the core insight
- Addresses a likely objection
- Sets up the next critical point
- Provides essential context for recommendations

## Visual Hierarchy

**Definition**: The most important information should be the most visually prominent.

### Hierarchy Principles

| Element | Should Be Most Prominent |
|---------|-------------------------|
| **Slide title** | Yes—it's the claim |
| **Key number or insight** | Yes—the single most important datum |
| **Supporting text** | Secondary |
| **Source notes** | Least prominent |
| **Decorative elements** | Should not compete |

### Evaluating Hierarchy

1. **Squint test**: Blur your eyes. What stands out? Is that the right thing?
2. **3-second test**: What do you register in 3 seconds? That should be the point.
3. **Print test**: Would this slide work printed on an A4 page in black and white?

### Common Hierarchy Failures

| Problem | Description |
|---------|-------------|
| **Competing elements** | Two things at equal prominence fighting for attention |
| **Buried lede** | Key insight is in small text at bottom |
| **Decorative dominance** | Graphics/icons draw eye away from content |
| **Wall of text** | Nothing stands out because everything is the same |
| **Chart overload** | 4 charts on one slide, none readable |

## CRAP Principles

Design fundamentals for slide layout (from Robin Williams' design principles):

### Contrast

- Different elements should be **noticeably different**
- Headers should be clearly distinct from body text
- Key numbers should pop
- If things aren't the same, make them very different

### Repetition

- Consistent patterns throughout the deck
- Same treatment for similar elements
- Builds visual rhythm and professionalism
- Headers, bullets, callouts should be styled consistently

### Alignment

- Every element should align with something else
- Avoid random placement
- Create strong invisible lines
- Left-align body text (avoid center-align for paragraphs)

### Proximity

- Related elements should be close together
- Unrelated elements should have clear separation
- Group labels with the data they describe
- White space creates groupings

### Quick Diagnosis

| Problem | Likely CRAP Violation |
|---------|----------------------|
| Slide feels busy but nothing stands out | Contrast |
| Slide feels chaotic, unprofessional | Alignment |
| Related items feel disconnected | Proximity |
| Every slide looks different | Repetition |

## Slide Type Standards

Different slide types have different standards:

### Title Slide
- Clear deck title
- Subtitle with context (client, date, project)
- Not overloaded with content

### Section Divider
- Section title tells a story (not just "Research Findings")
- Provides bridge from previous section
- Minimal content

### Content Slide
- Action title
- 3-5 bullets maximum
- One idea, clear takeaway

### Data Slide
- Chart earns majority of space
- Clear title stating the insight (not the metric)
- Annotation/callout highlighting the key point
- Source citation

### Summary Slide
- Synthesizes (doesn't just list)
- Limited to 3-5 key points
- Points are specific, not vague
- Leads to recommendations or next section

### Recommendation Slide
- Clear, actionable recommendations
- Prioritized (not a flat list)
- Connected to insights
- Includes owner/timing if relevant

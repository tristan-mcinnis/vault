# Section Structure

Detailed guidance for each proposal section.

## Executive Summary

**Purpose:** Give senior stakeholders the essence in 30 seconds.

**Length:** 3-5 sentences or a 3-column table.

**Format options:**

### Narrative
```markdown
## Executive Summary

A culturally-rooted brand positioning for Ballon Bleu in China, decoded through deep semiotics of the Circle (圆). We'll deliver an actionable framework combining expert-led semiotic analysis with focused consumer validation to identify ownable positioning territory that transforms Ballon Bleu from beautiful object into symbolic companion.
```

### Table
```markdown
## Executive Summary

| What We're Proposing | Why We're The Right Fit | Key Deliverables |
|---------------------|------------------------|------------------|
| [1-2 sentences] | [3 capability bullets] | [3 deliverable bullets] |
```

## 01. Context

**Purpose:** Establish the business situation and research need.

**Components:**

### Background
- Business challenge or opportunity
- Market context
- Why now?

### Strategic Imperative
> Single statement of what success looks like, formatted as blockquote.

### Objectives / Research Questions
- Organized into 2-4 clusters
- Phrased as questions, not statements

**Example:**
```markdown
### Research Question Clusters

#### 1. Competitive & Consumer Landscape
- Which brands own the milestone watch space?
- What emotional attributes do they own?

#### 2. Cultural & Semiotic Landscape
- What meanings does 圆 carry in contemporary China?
- Which facets resonate with young consumers?

#### 3. Brand Synthesis
- How do semiotic findings map to Ballon Bleu's design DNA?
```

## 02. Thought-starters

**Purpose:** Demonstrate IC's point of view before research.

**Quantity:** 2-4

**Structure per thought-starter:**
1. Provocative title
2. Observation/trend statement (bold)
3. Evidence (2-3 paragraphs, tables, examples)
4. Strategic question (blockquote with "How might we...")

See [[thought-starters]] for detailed format.

## 03. Methodology

**Purpose:** Explain how we'll answer the research questions.

**Required elements:**

### Approach Overview
Visual or table showing phases:
```markdown
| Phase | Duration | Focus |
|-------|----------|-------|
| Phase 1 | 4 weeks | Cultural Foundation |
| Phase 2 | 2 weeks | Consumer Validation |
| Phase 3 | 3 weeks | Synthesis & Delivery |
```

### Phase Details
For each phase:
- What we do (activities)
- Who we engage (sample)
- What we deliver (outputs)

### Rationale
Why this methodology, not just what.

## 04. Sample Design

**Purpose:** Specify who we're recruiting and why.

**Required elements:**
- Named profiles with Who/What/Why
- Summary breakdown table
- Recruitment criteria (include/exclude)
- City selection rationale (if multi-city)

See [[sample-design]] for templates.

## 05. Deliverables

**Purpose:** Set clear expectations for what client receives.

**Format:**
```markdown
## Deliverables

1. **[Deliverable Name]** — [Brief description, page count or format]
2. **[Deliverable Name]** — [Brief description]
3. **[Deliverable Name]** — [Brief description]
```

**Example:**
```markdown
1. **Positioning Territory Document** — Core narrative, messaging pillars, keyword lexicon (15-20 pages)
2. **Communication Themes Platform** — 3-4 thematic territories with channel applications (10-15 slides)
3. **Visual Cue Bank** — 30-50 curated references with usage guidance (digital toolkit)
```

## 06. Timeline

**Purpose:** Set expectations for project duration and key milestones.

**Formats:**

### Week-by-week
```markdown
| Week | Phase 1 | Phase 2 | Phase 3 |
|------|---------|---------|---------|
| W1 | Kick-off, desk research | — | — |
| W2 | Expert interviews | Recruitment | — |
| W3 | Stimulus development | Fieldwork | — |
| W4 | — | Fieldwork | Analysis |
| W5 | — | — | Reporting |
| W6 | — | — | Final delivery |
```

### Gantt-style
```markdown
| Phase | Dec W1 | Dec W2 | Dec W3 | Jan W1 | Jan W2 |
|-------|--------|--------|--------|--------|--------|
| Setup | X | X | | | |
| Fieldwork | | X | X | X | |
| Analysis | | | | X | X |
| Delivery | | | | | X |
```

## 07. Investment

**Purpose:** Transparent pricing with line-item breakdown.

See [[investment-tables]] for formats.

**Required:**
- Phase-by-phase breakdown
- Line items within phases
- Tax treatment stated
- Inclusions/exclusions

## 08. Team

**Purpose:** Introduce who will work on the project.

**Format:**
```markdown
## Team

| Role | Name | Expertise |
|------|------|-----------|
| **Project Lead** | [Name] | [2-3 lines on relevant experience] |
| **Lead Consultant** | [Name] | [2-3 lines] |
| **Recruitment Lead** | [Name] | [1-2 lines] |
```

For premium proposals, include individual team member pages with photos and detailed bios.

## 09. Relevant Experience

**Purpose:** Build credibility through past work.

**Format:**
```markdown
### [Client]: [Project Name]

**Challenge:** [1-2 sentences]

**Approach:** [2-4 bullets]

**Impact:** [1-2 sentences on outcomes]
```

**Selection criteria:**
- Same category (watches → watches, beauty → beauty)
- Same methodology (semiotics → semiotics)
- Same client (Cartier → Cartier past work)
- Similar business question

## 10. Contact

**Purpose:** Make it easy to move forward.

```markdown
## Contact

**Inner Chapter**

[Name] — [email]
[Name] — [email]
```

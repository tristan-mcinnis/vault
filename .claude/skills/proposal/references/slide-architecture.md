# Slide Architecture for Proposals

Mapping proposal sections to ic-slides archetypes and `[HINT:]` vocabulary. Use this reference when the target format is PowerPoint.

## Proposal Section → Archetype Mapping

These are recommendations, not mandates. The mapping guides `/pptx` toward appropriate visual treatments for each proposal section.

| Proposal Section | Likely Archetypes | Notes |
|---|---|---|
| Title/Cover | A1, A1c, A1d | Client name, project, date — billboard simplicity |
| Executive Summary | D5, I12 | 3-column "What / Why / What Gets" or narrative |
| Context/Background | C2, C10 | Strategic narrative — set the scene |
| Strategic Imperative | C1, C1b, C1d | Hero statement — billboard, single sentence dominates |
| Research Questions | I12, I13, D5 | Clustered questions as cards or columns |
| Thought-starters | C2, I14, C1 | POV + strategic question — one per slide |
| Methodology Overview | E2, E3 | Phase flow diagram — visual timeline |
| Phase Details | C2, I14 | What/why/outputs per phase |
| Sample Design | D5, I12, B2 | Profiles as character cards + quotas |
| Deliverables | G1, I12 | Numbered list as cards or iconography |
| Timeline | E2, F1 | Gantt or phase flow — visual timeline |
| Investment | F1, F2 | Line-item table (one of the few places tables are acceptable) |
| Team | B3, I12 | Grid of team members as character cards |
| Case Studies | C2, I14 | Challenge/approach/impact — one per slide |
| Contact | H2 | Closing slide — clean, minimal |

## Slide Count Guide (PPTX)

| Section | Slides | Notes |
|---------|--------|-------|
| Title | 1 | Project name, client, date, "Private & Confidential" |
| Executive Summary | 1 | 3-column or narrative format |
| Context | 2-4 | Background, objectives, research questions |
| Thought-starters | 4-8 | 1-2 slides per thought-starter |
| Methodology | 3-6 | Overview + phase details |
| Sample Design | 2-3 | Profiles + breakdown |
| Deliverables | 1 | Numbered list or cards |
| Timeline | 1 | Gantt or week-by-week |
| Investment | 1-2 | Table format |
| Team | 1-2 | Grid or individual pages |
| Case Studies | 2-4 | 1 slide per case |
| Contact | 1 | Team contacts |

**Total typical range:** 20-35 slides

## PPTX Hint Vocabulary for Proposals

Use conceptual hints (Section B vocabulary from ic-slides). These are the most common patterns for proposal content:

### Executive Summary
```
[HINT: iconography trio — 3 horizontal columns: "What We Propose" / "Why Now" / "What You Get", each with large icon + 2-line description — muted functional — no tables]
```

### Strategic Imperative
```
[HINT: hero quote — full bleed center, single strategic statement at billboard scale, client brand color accent — no competing visuals]
```

### Research Questions
```
[HINT: character cards — 3-4 horizontal columns, each card = question cluster with bold header + 2-3 questions below — muted functional — no more than 3 bullets per card]
```

### Thought-starters (Setup Slide)
```
[HINT: hero quote — bold observation statement centered large, evidence summary below in small type — high contrast stark — quote occupies 50%+ of slide]
```

### Thought-starters (Strategic Question Slide)
```
[HINT: hero quote — full bleed center, "How might we...?" at billboard scale, warm golden hour — no competing visuals]
```

### Methodology Overview
```
[HINT: visual timeline — 3-4 phases flowing left to right, each phase with icon + name + duration, connecting arrows between — muted functional — no tables]
```

### Phase Detail
```
[HINT: split 30/70 — left column: phase name + duration badge, right column: 3 bullets (what/who/output) — muted functional — no more than 3 bullets]
```

### Sample Profiles
```
[HINT: character cards — 3 horizontal columns, each card with color-coded header, profile name, "Who / What / Why" as subheads — warm golden hour — no tables]
```

### Sample Breakdown
```
[HINT: iconography trio — 3-4 hero numbers (total n, cities, segments), each with icon + label — high contrast stark — no tables]
```

### Deliverables
```
[HINT: ascending step-ladder — numbered deliverables flowing top to bottom, each with name + one-line description — muted functional — no competing visuals]
```

### Timeline
```
[HINT: visual timeline — weeks flowing left to right, phase bars showing duration, key milestones marked — muted functional — no tables]
```

### Investment
Investment is one of the few sections where a table IS the right encoding:
```
[HINT: data — clean table with phase rows, line items, subtotals, grand total highlighted — high contrast stark]
```

### Team
```
[HINT: character cards — 2x2 or 2x3 grid, each card with photo placeholder + name + role + one-line expertise — muted functional — no more than 3 bullets per card]
```

### Case Studies
```
[HINT: split 30/70 — left: client logo area + project name, right: Challenge (1 line) / Approach (3 bullets) / Impact (1 line) — high contrast stark — no competing visuals]
```

### Contact
```
[HINT: hero quote — company name centered large, team contact details below in clean columns — muted functional — no competing visuals]
```

## Section Dividers

Use section dividers between major acts. In proposals, dividers typically appear before:
- Context (after title/exec summary)
- Thought-starters
- Methodology
- Investment
- Team / Case Studies

```
[HINT: section divider — section number + name, navy background or left border accent — deep luxurious]
```

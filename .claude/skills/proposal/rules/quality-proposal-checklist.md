# Proposal Quality Checklist

Validate before moving from draft to render mode.

## Required Checks

### Executive Summary
- [ ] Captures the "so what" in 3-5 sentences
- [ ] Senior stakeholder can understand value in 30 seconds
- [ ] Includes what we're proposing, why it matters, what client gets

### Objectives
- [ ] Written as questions, not statements
- [ ] Grouped into 2-4 logical clusters
- [ ] Each cluster has clear strategic focus

### Thought-starters
- [ ] Each ends with "Strategic Question: How might we...?"
- [ ] Shows IC's point of view, not neutral observation
- [ ] Includes evidence/data where available
- [ ] 2-4 thought-starters (not more)

### Methodology
- [ ] Clear phase structure (typically 2-4 phases)
- [ ] Each phase has defined outputs
- [ ] Rationale for methodology choices included
- [ ] Final phase includes synthesis approach

### Sample Design
- [ ] Profiles explain "why we need them" (not just demographics)
- [ ] Named personas (e.g., "The Aspiring Collector")
- [ ] Clear quotas by dimension (city, segment, demographics)
- [ ] Screening criteria as bullet list

### Investment
- [ ] Line-item breakdown by phase
- [ ] Shows what's included/excluded
- [ ] Tax treatment stated
- [ ] Optional add-ons separated if applicable

### Timeline
- [ ] Week-by-week or Gantt-style format
- [ ] Realistic with buffer built in
- [ ] Key milestones marked
- [ ] Dependencies clear

### Relevant Experience
- [ ] 2-4 case studies included
- [ ] Cases relevant to this project type
- [ ] Shows IC's track record in similar work

## Avoid

| Pattern | Problem |
|---------|---------|
| Skipping executive summary | Senior stakeholders won't read further |
| Objectives as statements | "Understand..." isn't actionable |
| Generic methodology description | Doesn't show why this approach fits |
| Sample profiles as just demographics | Misses the strategic "why" |
| Round-number budgets only | Looks like a guess, not a plan |

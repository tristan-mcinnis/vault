# VF Corporation Standards

For VF brand projects (Vans, TNF, Timberland, Icebreaker), apply these formatting requirements.

## Metadata Header

Insert at top of every VF transcript:

```
**Transcript Structuring & Documentation Standards**

Date: [Interview/FG date]
Brand: [VF brand name]
Research Type: [Product feedback / Store feedback / Trend exploration]
Tested Element: [Specific product if applicable]
Market / Country: [Mainland China, Taiwan, Korea, Australia, etc.]
City Tier: [Tier 1 / Tier 2 / Tier 3, if applicable]
Consumer Profile: [VF buyer/Non-buyer, Competitor brands, Gender, Age]
```

## Product Callouts

For multi-product sessions, insert section headers:

```
--- VANS SS26 Super Low Pro - White ---
```

Include: product name/code, season, color.

## File Naming Convention

Pattern: `XX brand XX market XX season XX category/product/study - XX YEAR XX Month`

| Type | Example |
|------|---------|
| Product feedback | `TNF China FW25 Hardshell - 2025 April` |
| Store feedback | `Vans Taiwan SS26 Store Feedback - 2025 June` |

## Avoid

| Pattern | Problem |
|---------|---------|
| Missing metadata fields | Incomplete context for analysis |
| Product callouts without specifics | Can't track which product feedback applies to |
| Generic file names | Hard to locate and organize transcripts |

# Theme Development Reference

Identify and develop themes from qualitative research data through systematic analysis.

## Theme vs. Code

| Concept | Definition | Level |
|---------|------------|-------|
| **Code** | Label for a specific piece of data | Granular, close to data |
| **Theme** | Pattern of meaning across codes | Abstract, interpretive |

Themes are built from codes. Multiple codes may contribute to a single theme.

## Theme Identification Process

### 1. Immersion
- Read all data multiple times
- Note initial impressions and patterns
- Don't rush to conclusions

### 2. Code Clustering
- Group related codes together
- Look for underlying connections
- Ask: "What unifies these codes?"

### 3. Theme Articulation
- Name the pattern
- Write a clear theme statement
- Test if it captures the essence

### 4. Theme Refinement
- Check theme against data
- Ensure all data within theme fits
- Split or merge as needed

### 5. Theme Mapping
- Identify relationships between themes
- Build hierarchical structure
- Note tensions and contradictions

## Theme Quality Criteria

**Strong Themes**:
- **Distinctive**: Clearly different from other themes
- **Coherent**: Internal consistency across data points
- **Substantive**: Meaningful pattern, not trivial observation
- **Grounded**: Supported by multiple data points
- **Interpretive**: Goes beyond description to meaning

**Weak Themes**:
- Too broad (captures everything)
- Too narrow (describes single data point)
- Merely descriptive (restates data)
- Forced (doesn't fit data naturally)
- Overlapping (indistinguishable from another theme)

## Theme Types

| Type | Definition | Example |
|------|------------|---------|
| **Convergent** | Pattern appearing across sources | "Trust is built through consistency" |
| **Divergent** | Contradiction or tension | "Price sensitivity vs. quality demands" |
| **Emergent** | Unexpected finding | "Social media as decision validator" |
| **Confirming** | Expected pattern verified | "Convenience drives channel choice" |

## Theme Statement Format

A theme statement should be:
- **Complete sentence**: Not just a topic label
- **Interpretive**: Conveys meaning, not just description
- **Specific**: Captures what the data actually shows

| Bad Example | Good Example |
|-------------|--------------|
| "Price" | "Price sensitivity increases when differentiation is unclear" |
| "Customer service" | "Service recovery can create stronger loyalty than flawless service" |
| "Quality concerns" | "Quality is assumed until proven otherwise; single failures create lasting doubt" |

## Thematic Framework Development

### Building Theme Hierarchy

```
Overarching Narrative
├── Major Theme 1
│   ├── Sub-theme 1a
│   └── Sub-theme 1b
├── Major Theme 2
│   ├── Sub-theme 2a
│   └── Sub-theme 2b
└── Major Theme 3 (Tension/Contradiction)
```

### Theme Relationships

| Relationship | Description |
|--------------|-------------|
| **Hierarchical** | Theme contains sub-themes |
| **Parallel** | Themes exist independently |
| **Tension** | Themes contradict or complicate each other |
| **Sequential** | One theme leads to another |
| **Causal** | One theme explains another |

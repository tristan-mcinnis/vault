# Quality Litmus Test for Insights

Use this checklist to validate L3 insights before including in deliverables.

## The Test

| # | Question | Pass Criteria |
|---|----------|---------------|
| 1 | Identifies strategic tradeoff? | Clear DO/DON'T articulated |
| 2 | Explains what we WON'T do? | Explicit rejection stated |
| 3 | Would competitor feel threatened? | Creates defensible advantage |
| 4 | Challenges category assumption? | Goes beyond obvious |
| 5 | Has "instead of" statement? | Alternative path rejected |
| 6 | Business impact quantified? | Numbers or directional estimate |

## Scoring

- **6/6**: Strong insight, ready for final deliverable
- **4-5/6**: Needs refinement before delivery
- **<4/6**: Still at L1/L2 level, elevate further

## Application

Run this test on every insight before including in:
- Client presentations
- Final reports
- Strategic recommendations

## Avoid

| Pattern | Problem |
|---------|---------|
| Skipping validation | Weak insights in final output |
| Passing insights that fail 2+ criteria | Dilutes quality |
| Forcing pass on weak insights | Erodes credibility |

## Do Instead

| Pattern | Benefit |
|---------|---------|
| Test every insight | Consistent quality |
| Send back insights that fail | Forces elevation |
| Document which criteria failed | Guides improvement |

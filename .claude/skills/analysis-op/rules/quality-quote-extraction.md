# Quote Extraction Standards

Verbatim quotes provide evidence and bring findings to life. Extract with proper attribution.

## Required Elements

| Element | Standard | Example |
|---------|----------|---------|
| **Quote text** | Verbatim, preserve original language | "I just want something that feels like mine" |
| **Speaker** | Participant identifier | P3, Participant 7, Expert 2 |
| **Session** | Session/interview reference | IDI Session 2, FGD Group A |
| **Timestamp** | If available from transcript | 23:45 |
| **Theme** | Finding/insight it supports | Personalization need |

## Format

```markdown
> "I just want something that feels like mine, not like everyone else's." - P3, IDI Session 2, 23:45 | Theme: Personalization need
```

Attribution comes **after** the closing quotation mark, separated by ` - ` (space-hyphen-space). Do NOT place attribution inside the quotes.

## Selection Criteria

Prioritize quotes that:
- Articulate the insight in the participant's own words
- Reveal underlying motivation (why, not just what)
- Are vivid, memorable, quotable
- Represent a pattern (not outlier)
- Could be used in client presentations

## Language Preservation

| Do | Don't |
|----|-------|
| Keep original phrasing | Clean up grammar |
| Preserve hesitations if meaningful | Remove all filler words |
| Include [laughter], [pause] if relevant | Over-notate |
| Use [...] for omissions | Change meaning through omission |

## Quote Density

| Output Type | Recommended |
|-------------|-------------|
| Findings pack | 2-3 quotes per finding |
| Insight card | 1-2 signature quotes |
| Final report | 1 anchor quote per theme |

## Avoid

| Pattern | Problem |
|---------|---------|
| Paraphrasing as quote | Not verbatim evidence |
| Missing attribution | Can't verify source |
| Quote without context | Meaning unclear |
| Too many quotes | Dilutes impact |

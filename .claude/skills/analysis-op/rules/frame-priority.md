# Priority Framework

Classify recommendations by impact and feasibility to guide implementation sequencing.

## Priority Matrix

| Priority | Definition | Timeline | Action |
|----------|------------|----------|--------|
| **P1** | High impact + High feasibility | 0-3 months | Execute immediately |
| **P2** | High impact + Low feasibility | 3-12 months | Plan and resource |
| **P3** | Low impact + High feasibility | Fill-ins | Do when capacity allows |
| **P4** | Low impact + Low feasibility | Deprioritize | Revisit later or drop |

## Assessment Criteria

**Impact** (High/Low):
- Revenue/margin potential
- Customer experience improvement
- Competitive differentiation
- Strategic alignment

**Feasibility** (High/Low):
- Resource availability
- Technical complexity
- Organizational readiness
- Time to implement

## Application

1. Score each recommendation on impact (H/L) and feasibility (H/L)
2. Assign priority based on matrix
3. Group recommendations by priority in final deliverable
4. Ensure P1s have clear owners and timelines

## Avoid

| Pattern | Problem |
|---------|---------|
| All P1s | Lack of prioritization |
| No P4s | Not making hard choices |
| Priority without rationale | Arbitrary ranking |

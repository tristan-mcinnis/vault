# Presentation Shapes

Before writing any report or debrief, identify the project type and apply the corresponding narrative shape. The shape determines the ORDER of argument — not just the content.

**Core principle:** A debrief is a perspective, not a process. Freshness, structure, and a point of view matter more than comprehensiveness. Quality not quantity. Tell a story, have an opinion.

## The Four Project Types

| Project Type | Narrative Logic | Output Feels Like |
|---|---|---|
| **Market Entry** | Cultural context → brand pillars → activation (Message / Product / Retail / Media / Audience) | A business plan document |
| **Brand Positioning** | Fixed brand/product → cultural insights + market context → category → competitors → positioning → territory/statement/guardrails | A sales pitch for your idea |
| **Creative / Concept** | Context/filters → answer in summary → what didn't work → what worked + filters → concept coverage → moving forward | A creative brief with a clear verdict |
| **Audience Understanding** | Overall audience intro → foundational shifts/challenges → segments (one by one: portrait, psychographic, values, relationship, opportunities) → business/brand filters → implications by touchpoint | A good movie script — vivid, immersive, character-driven |

## How to Identify the Project Type

| If the client brief is about... | Project type |
|---|---|
| Entering a new market or launching a brand in a new country | Market Entry |
| Repositioning a brand, finding a new territory, or owning a space | Brand Positioning |
| Testing product concepts, ad concepts, or creative routes | **Creative / Concept** |
| Understanding a target consumer segment or audience | Audience Understanding |

Most qualitative research projects in FMCG/CPG fall into **Creative/Concept** or **Audience Understanding**.

## Creative / Concept Shape

For concept testing, innovation research, and creative evaluation:

```
1. Context/filters        — business situation, brief, product truth, who we spoke to (brief, 2-3 slides max)
2. Answer in summary      — lead with the verdict upfront. Don't build to it.
3. Consumer landscape     — need states, occasions, or frameworks that explain WHY concepts win/lose
4. Brand & competitors    — competitive context grounded in the framework above
5. What didn't work, why  — concepts that failed + clear failure mode
6. What worked            — key filters/principles that explain what wins
7. Coverage of each route — per-concept deep dive (scores, evidence, improvements)
8. Implications           — portfolio architecture, dual-track if applicable, next steps
```

**Key differences from encyclopedic structure:**
- Answer comes SECOND (slide 2), not after all the evidence
- "What didn't work" comes BEFORE "what works" — establishes the evaluative filter
- Context is brief, not a full methodology section
- No separate "strategic implications" section — implications are embedded in each concept

### Product Truth in Context

Within Context (step 1), establish the **product truth** — what the brand currently delivers to its loyalists. This anchors the evaluation: concept testing must be grounded in what you're building *from*, not just what you're building *toward*. Without product truth, the audience can't judge whether a concept extends the brand or abandons it.

*Learned from: Chips Ahoy Premium Innovation report (Mar 2026). Lisa asked why current-user loyalty wasn't established before showing gaps. Adding "crunch-then-melt sensory + nostalgic reliability" as product truth grounded the entire concept evaluation.*

### Framework Introduction: Earn Your Axes

When introducing a proprietary analytical framework (2x2 matrix, segmentation, need-state map), you must **bridge from the data to the framework** before deploying it. Don't jump into dimensions — motivate *why those dimensions matter*.

**Pattern:**
1. Show the raw observation that demands a framework (e.g., "mood uplift means different things to different people")
2. Bridge: preview where the framework is heading in one sentence
3. Introduce Axis 1 with evidence
4. Introduce Axis 2 with evidence
5. Deploy the full framework

**Build-up vs. overview-first:** Build-up (axis → axis → full framework) creates an aha moment and earns conviction. Overview-first (full framework → explain each cell) creates immediate clarity but loses the discovery. Default to build-up for live presentations; consider overview-first for leave-behind decks.

*Learned from: CA Premium report. Lisa flagged that the Consumer Landscape section jumped into the energy/need 2x2 without motivating why those axes. Adding a bridging sentence before the energy axis resolved the logic gap.*

### Multi-Winner Portfolio Implications

When multiple concepts win for different segments or occasions, the Implications section needs:
1. **Mission clarity** — each winning concept paired with its target user and need state
2. **Cannibalization check** — show that the concepts serve different consumer jobs (not the same buyer in the same moment)
3. **Portfolio architecture** — side-by-side comparison of base product + new concepts showing distinct targets, missions, formats, and price points

*Learned from: CA Premium report. C3 (Hazelnut) won for current-user trade-up in the Reward need state; C4 (Brownie) won for lapsed-user recruitment in the Indulge need state. The portfolio slide showing three distinct products with three distinct jobs was essential for landing the dual-track recommendation.*

## Brand Positioning Shape

```
Fixed brand/product (anchor)
  → Cultural insights + market context
    → Category dynamics
      → Competitors
        → Positioning recommendation
          → Territory / Statement / Guardrails / Visualisation
```

Write this as a sales pitch for your idea. One right answer — be confident and precise.

## Audience Understanding Shape

```
1. Introducing the overall audience (quant data, hypotheses, personas)
2. Foundational shifts / challenges
3. The segments — one by one:
   - Portrait (who they are)
   - Psychographic / behavioral
   - Values / attitudes
   - Relationship with brand
   - Opportunities
4. Business/brand filters
5. Implications: priorities across Product / Retail / Media per segment
```

Write this like a good movie script — vivid, immersive, character-driven. Details bring things to life. Avoid clichés and stereotypes. Show how brand behavior would need to change based on what you learned.

## Market Entry Shape

```
Cultural context (filter 1)
  → Brand pillars (filter 2)
    → Activation implications:
       Message / Product / Retail / Media / Audience
```

Used as long-term planning documents. Think short-term wins vs. long-term differentiation. Client often doesn't know the market — cultural context is essential and valuable.

## Narrative Rhythm: "What Is" / "What Could Be"

Nancy Duarte's analysis of great presentations found they share an oscillating structure — pushing the audience between the painful current reality and the aspirational future state. This rhythm overlays any of the four shapes above.

```
What Is (tension)  →  What Could Be (relief)  →  What Is (deeper tension)  →  What Could Be (bigger vision)
```

In practice for research presentations, this translates to:

| "What Is" (tension) | "What Could Be" (relief) |
|---|---|
| The base product is losing 18-25s | A premium line captures the trade-up occasion |
| Consumers say "too sweet, childish" | "Earned indulgence" framing removes the ceiling |
| Competitor X owns the ritual space | Our texture story creates a new ritual territory |
| Concept A failed — wrong occasion | Concept B wins the exact occasion we need |

**How to apply it:**
- Don't stack all the problems first and all the solutions last. Interleave them.
- After showing a painful finding, immediately follow with the opportunity it reveals.
- The oscillation creates emotional momentum — tension-relief-tension-relief — that keeps the room engaged far more than a linear march from problems to recommendations.
- Section dividers are natural oscillation points: end an act on tension, open the next on possibility.

**This does NOT replace the presentation shapes.** The shapes determine argument order and logic. The oscillation determines emotional rhythm within that order. A Creative/Concept shape still leads with the verdict — but the body of the deck oscillates between what's failing and what's winning.

## Structural Principles for Full Reports

These principles apply across all presentation shapes. They govern how frameworks, findings, and arguments hold together across an entire deck.

### Introduce → Explain → Resolve

Every framework, segmentation, or typology you introduce must be **resolved** somewhere in the report. Resolution doesn't mean every element maps to a concept or recommendation — it means the audience understands what happened to each piece.

**Three valid resolutions:**
- **"This is where we win"** — the element maps directly to a winning concept or opportunity
- **"This is where we lose"** — the element explains a gap, loss, or competitive disadvantage
- **"This isn't where the fight is"** — the element is acknowledged and narrowed out, focusing the argument

**The anti-pattern is silence.** If you introduce four need states and only two appear in the concept evaluation, the audience spends the rest of the deck wondering about the other two. You don't need to evaluate concepts against all four — but you need to *resolve* the other two, even if the resolution is a single slide showing "CA loses here, so the battlefield is over here."

**Pattern:**
```
Introduce (here are 4 need states)
  → Explain (here's what each one looks like)
    → Resolve ALL (CA loses in 2, so the fight is in the other 2)
      → Evaluate concepts against the relevant ones only
```

*Learned from: CA Premium report. Four need states introduced (Recharge, Refresh, Reward, Indulge). Recharge and Refresh resolved as "where CA loses" (slide 19). Concepts then evaluated only against Reward and Indulge — the audience accepted this because the other two were resolved, not abandoned.*

### Inductive Discovery → Deductive Narrative

Research is inductive: you run fieldwork, patterns emerge, and you figure out why things worked or didn't. But the report should read as **deductive**: here are the principles, now watch them predict the results.

This is the most important structural conversion in reporting. The audience doesn't need to relive your discovery process — they need to receive your conclusions as a logical argument.

**How it works in practice:**

| What actually happened (inductive) | How the report presents it (deductive) |
|---|---|
| We tested 5 concepts. C3 won, C1 failed. We noticed C3 added texture layers while C1 swapped ingredients. | "Three principles predicted every concept result" → Principle 1: texture layers beat ingredient swaps → C3 passes, C1 fails |
| Consumers talked about energy levels and "I need" vs "I deserve." We realised these were two axes. | "Mood uplift fragments into 4 need states" → here's the 2x2 → here's what each quadrant looks like |
| We saw that health messaging killed C1's appeal across all groups. | "Health messaging destroys indulgence" → presented as a universal rule, then demonstrated with C1 evidence |

**The conversion:** Extract the evaluative logic from your results, elevate it into principles, and position those principles *before* the detailed evidence — even though you discovered them after. The report reads as "here are the rules → here's how they apply" rather than "here's what happened → here's what we think it means."

**Why this matters:** A deductive narrative gives the client a *toolkit* — principles they can apply to future decisions beyond this specific research. An inductive narrative gives them a *diary* of what you found. The toolkit is more valuable.

*Learned from: CA Premium report. The "three underlying principles" slide (slide 6) presents rules that were actually extracted from concept testing results. Positioned early, they frame the entire deck — the audience pre-agrees with the logic, then concept verdicts feel inevitable rather than opinionated.*

### Competitive Embedding

Rather than isolating competitive analysis in a standalone section, **embed competitive context inside the frameworks where it matters** — then use the standalone section as a summary.

**Pattern:**
```
Need state detail page → include "who wins here" for each need state
  → Audience absorbs competitive context in situ
    → Standalone competitive section becomes CONFIRMATION, not new information
```

**Why this works:** By the time the audience sees the full competitive table, they've already encountered each competitor in the context where it matters. The table confirms and organises what they've been absorbing — it doesn't surprise them.

**The anti-pattern** is a competitive section that feels disconnected: "now let's talk about competitors" with no link to the framework or need states that preceded it. The audience can't evaluate competitive threats without knowing *which occasion* the competitor wins.

*Learned from: CA Premium report. Each need-state detail page (slides 14-17) included "Who wins here" and "CA gap." The standalone Brand & Competitors section (slides 24-28) then summarised and deepened what was already planted.*

## Anti-Patterns to Avoid

| Anti-pattern | Why it fails |
|---|---|
| Encyclopedic A-E sections | Topics, not arguments. Buries the answer. |
| Methodology-first | Front-loading process signals insecurity, not authority |
| Separate "implications" section | Implications belong with each finding, not at the end |
| Building to the answer | Senior leaders want the verdict first, evidence second |
| Equal time for all concepts | Winners deserve more space than losers |
| Deploying a framework without earning the axes | Audience rejects the framework because they don't know why those dimensions were chosen |
| Need states described only with emotions | Without behavioral markers, frameworks feel theoretical — R&D and marketing can't act on feelings alone |
| Section dividers without connective logic | Each section feels like a separate presentation instead of one argument |
